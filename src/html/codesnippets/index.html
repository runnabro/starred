CodeSnippets

Making it easy to find, run, and share code. Work done with Sass, Backbone.js, & Handlebars.

View It Live

When I joined the team, Runnable was already a functional product (now CodeSnippets). Our goal was to help developers find, run, and share their code.

Rethinking Our Homepage

Our homepage was focused on helping users discover code, but it required our users to search, or browse by language (or framework) before being able to see code. We went to work redesigning this page with two goals in mind: increasing user created content, and return visits.

Creating Content

On our first homepage, the sole path to creating a code example is with the “Create” button in the header. We were consistently getting feedback that users didn’t know what languages we supported.

To make it so users were just one click from getting the code in their language running, we exposed our create flow front and center.

Stickiness

The paths to finding code examples required users to either search or click through categories. We wanted the value we provided the be immediately tangible and feel “fresh”.

To encourage return visits and reduce the gap to seeing running code, we implemented a feed for new and popular content. We also generated feed pages for every tag (including user submitted ones!).

User Profiles

We added profiles to each code snippet for the same reasons for rethinking our homepage. Users were able to use their profile to keep track of their code snippets and their profile would also appear on their contributions.

Build & Runtime

One of the defining features of CodeSnippets is that users have access to their environment. We expanded on that by giving users the ability to execute commands before the environment is launched and to define how their environment is launched.

Refactoring for Mobile

When I joined the team, our userbase sat at ~70% desktop users. Today, we’re looking at 70%+ mobile users. Throughout the months of active development our pivot, I slowly rewrote CodeSnippets to be more semantic, accessible, and responsive.
