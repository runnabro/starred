Rethinking Our Homepage

Our homepage was focused on discovering code examples, but it required our users to search, or browse by language (or framework) before being able to see code. We rethought our page with two primary goals:

  1. Acquisition. In our new design, users would be able to see and start creating new snippets from the hero. Previously, users needed to find the “Create” call-to-action in the header.

  2. Stickiness. Below the hero, our page was static, and then two clicks removed from viewing code. We implemented a feed for new and popular content to increase daily usage with content that our users could access in a single click.

User Profiles

We added profiles to each code snippet for the same reasons for rethinking our homepage. Users were able to use their profile to keep track of their code snippets and their profile would also appear on their contributions.

Build & Runtime

One of the defining features of CodeSnippets is that users have access to their environment. We expanded on that by giving users the ability to execute commands before the environment is launched and to define how their environment is launched.
