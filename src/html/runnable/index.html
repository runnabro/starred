Working With Code, Not Builds

When exploring how to show running code to users we noticed many CI tools listed code by build number, but identifying builds by number makes it hard to identify what code is running. By showing code by the commits and branches a developer creates, we are designing our product for the developer workflows.

Our users loved how accessible their environments were with our GitHub integration. We wanted to provide the same benefits for our PM and QA users by creating an integration for Jira as well.
