Product Marketing

Before starting with product, prospective users are funneled to our marketing page. It’s one of the best opportunities we have to convince them to give our product a shot! I collaborated with our product manager and the other designer to work on everything design and more; including copywriting and writing Javascript for the interactions we needed. Our current version is the result of dozens of iterations as our product and marketing strategy has evolved.

Making Docker Easy

Teams trying Runnable were also switching to Docker. Since Docker is an important part of getting your code running, we made it easy to define environment. Users who didn’t have a Dockerfile yet, would be able to create a template based on their environment needs.

After making the necessary configuration for their environment, users would have a working Dockerfile.

Building With Branches

When exploring how to show running code to users we noticed many CI tools listed code by build number, but identifying builds by number makes it hard to identify what code is running. By showing code by the commits and branches a developer creates, we are designing our product for the developer workflows.

Bringing Environments to the Users

Our users loved how accessible their environments were with our GitHub integration. We wanted to provide the same benefits for our PM and QA users by creating an integration for Jira as well.

Our integrations with the developer workflow meant that our users would only need to enter the app to make configuration changes or to debug issues. We laid out our UI to make it easy for users to find their environments, and access build information quickly.
